ID Mapping
=============================
Virtual Socket Manager vs_main is assigned ID 0
  Reconfigurable Module rm_add is assigned ID 0
  Reconfigurable Module rm_comp is assigned ID 1
  Reconfigurable Module rm_div is assigned ID 2
  Reconfigurable Module rm_inc is assigned ID 3
  Reconfigurable Module rm_dec is assigned ID 4
  Reconfigurable Module rm_mod is assigned ID 5
  Reconfigurable Module rm_mul is assigned ID 6
  Reconfigurable Module rm_mux is assigned ID 7
  Reconfigurable Module rm_reg is assigned ID 8
  Reconfigurable Module rm_shl is assigned ID 9
  Reconfigurable Module rm_shr is assigned ID 10
  Reconfigurable Module rm_sub is assigned ID 11
  Reconfigurable Module RM_12 is assigned ID 12


Address Structure
=============================
The addresses are structured as follows:
  There is no VSM Select slice as there's only one VSM
  address[10: 9]  : Selects the Register Bank
  address[ 8: 2]  : Selects the Register within the bank
